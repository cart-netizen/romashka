    # Инструкция по наполнению сайта «Ромашка» (Directus)

Весь контент сайта ведётся в админке **Directus**. Программисты не нужны —
всё делается через визуальный интерфейс.

- **Локально:** http://localhost:8055
- **Прод:** https://cms.<ваш-домен> (например `cms.romashka.ru`)
- Вход — по e-mail и паролю администратора.

## Главные правила (важно!)

1. **Статус = «Опубликовано».** Любая запись (товар, категория, отзыв…) видна на
   сайте только при статусе **Опубликовано**. Черновики и архив не показываются.
2. **Обновление не мгновенное.** Сайт кэширует данные — изменения появляются в
   течение ~2 минут. Это нормально.
3. **Фото каталога** загружайте прямо в поле товара/категории/фабрики — они
   автоматически попадают в публичную папку и показываются на сайте.
4. **Цена — «от ___ ₽».** В поле «Цена от» вводите число в рублях (например
   `189000`) — на сайте отобразится «от 189 000 ₽».

## Рекомендуемый порядок наполнения

1. Настройки сайта → 2. Категории → 3. Подкатегории → 4. Цвета → 5. Фабрики →
6. Товары → 7. Промо-плитки меню, USP, отзывы → 8. Сцены «Shop the look» →
9. Материалы → 10. Дизайнеры → 11. Сделки.

---

## 1. Настройки сайта (Site Settings)

Меню слева → **Site Settings** (одна запись). Заполните:
- Телефон, E-mail, Часы работы, Адрес салона.
- Ссылка на мессенджер MAX, ссылки на соцсети.
- Карта (map_embed) — вставьте код `<iframe>` Яндекс/Google Карт или URL.
- Дисклеймер размеров, дефолтный срок изготовления.
- Сумма промокода (5000), префикс промокода (например `ROMASHKA`).
- SEO: заголовок/описание по умолчанию, тексты hero.

## 2. Категории (Categories)

Создайте 4: Диваны, Кровати, Кресла, Тумбочки.
- **Название**, **Слаг** (латиницей: `sofas`, `beds`, `armchairs`, `nightstands`),
  Описание, **Изображение** (hero), Сортировка, Статус = Опубликовано.

## 3. Подкатегории (Subcategories)

Для каждой категории — её виды. Пример (Диваны): Угловые, Прямые, Двухместные,
Трёхместные, Модульные, Кожаные.
- Название, Слаг, **Категория** (выбрать родителя), Сортировка, Статус.

## 4. Цвета (Colors)

Свотчи для фильтра и карточек: Название + **HEX** (например `#E8DCC8`).

## 5. Фабрики (Factories)

- Название, Слаг, **Логотип/фото**, Описание, Сайт, Сортировка, Статус.

## 6. Товары (Products) — главное

Создать товар → заполнить:
- **Название**, **Слаг** (латиницей, уникальный), **Артикул** (sku).
- **Категория**, **Подкатегория**, **Фабрика**.
- **Цена от** (число, ₽).
- Краткое описание (для карточки в сетке), Описание (вкладка «Информация о товаре»).
- **Главное фото** (main_image) и **Галерея** (несколько фото).
- **Каркас** (Массив/Металл/Фанера), **Обивка** (несколько вариантов), **Цвета**.
- **Характеристики** (повторяющийся блок: Параметр → Значение): Каркас, Обивка,
  Наполнение, Ножки, Уход… — это таблица «Материал изделия и уход».
- **Размеры** (повторяющийся блок: Название + Ширина/Высота/Глубина в см) — кнопки
  выбора размера на карточке.
- **Чертежи** (dimensions_images) — для блока «Фактические размеры».
- Срок изготовления (если отличается от дефолтного).
- Габариты Ш/В/Г (для фильтра по размерам).
- **Флаги:** Хит (is_bestseller), Новинка (is_new), Распродажа (is_sale),
  В наличии (in_stock).
- Сортировка, **Статус = Опубликовано**.

## 7. Промо-плитки меню, USP, отзывы

- **Menu Promos** — плитки справа в выпадающем mega-menu (Новинки/Хиты/Распродажа).
  Поля: **Категория** (плитка покажется ТОЛЬКО в меню этой категории), **Заголовок**,
  **Изображение** (своё фото плитки), **Бейдж** (выпадающий список:
  Новинки / Хиты коллекции / Распродажа), Сортировка, Статус.
  Для каждой категории заведите свои 3 плитки и выберите в **Бейдж** нужное значение —
  ссылка соберётся автоматически (плитка откроет товары этой категории с этим бейджом).
  Поле **Ссылка** обычно оставляйте пустым (нужно только чтобы переопределить адрес вручную).
  Сами бейджи на товарах ставятся в **Products** галочками **Новинка** (`is_new`),
  **Хит** (`is_bestseller`), **Распродажа** (`is_sale`); товар должен быть в нужной
  **Категории** и **Опубликован**.
- **Hero-баннер каталога** (узкое фото с заголовком вверху страницы категории) берётся
  из самой **категории**: поле **Hero image** (фото), **Название** и **Описание**
  (текст поверх фото). Без фото — бордовый фон с тем же текстом.
- **USP Messages** — ротация сообщений на карточке товара: Текст, Иконка, Статус.
- **Reviews** — кураторские отзывы: выбрать **Товар**, Имя автора, Рейтинг (1–5),
  Текст, Статус = Опубликовано. (Публичной регистрации отзывов нет.)

## 8. «Shop the look» (Showcase Scenes + Hotspots)

- **Showcase Scenes** — до 5 фото-сцен интерьера: Заголовок, Изображение, Статус.
- **Showcase Hotspots** — маркеры на сцене: выбрать **Сцену** и **Товар**, задать
  **pos_x** и **pos_y** в процентах (0–100, положение точки на фото). На сайте —
  пульсирующий маркер с названием и ценой, клик ведёт на товар.

  **Координаты не угадывайте — берите их кликом.** Откройте служебную страницу
  **`https://<домен>/scene-editor`** (например `https://paleron.ru/scene-editor`;
  она не индексируется и не показана в меню — заходите по прямой ссылке):
  1. выберите **сцену** в списке;
  2. **кликните по фото** в нужной точке;
  3. внизу появятся готовые **pos_x** / **pos_y** (кнопка «Копировать» кладёт оба
     числа через таб);
  4. впишите их в поля **pos_x** / **pos_y** новой записи **Showcase Hotspots**
     (там же выберите Сцену и Товар).

  Геометрия пикера (16:9 с обрезкой по краям) совпадает с сайтом, поэтому маркер
  встаёт ровно туда, куда кликнули. Тёмные точки на фото — уже существующие
  маркеры (для ориентира). Чтобы сцена появилась в пикере, она должна быть
  **Опубликована** и иметь изображение.

## 9. Материалы (Materials) — для кабинета дизайнеров

Прайсы, чертежи, фактуры, условия. Поля: Название, Описание, **Файл** или **Ссылка**,
**Категория** (прайсы/чертежи/фактуры/условия/маркетинг), Статус.
Файлы материалов **приватные** — видны только дизайнерам в кабинете.

## 10. Дизайнеры (пользователи)

Меню → **User Directory** → New User:
- Имя, Фамилия, **E-mail** (реальный TLD), **Пароль**, **Роль = Designer**,
  Статус = Active.
Дизайнер войдёт на сайте по адресу `/cabinet/login` и увидит **только свои** сделки
и материалы.

## 11. Сделки (Deals) — как заносить

Меню → **Deals**. Удобно вести в виде **Kanban** (колонки по статусу — перетаскивайте
карточки). Поля сделки:
- **Номер**, **Название** (например «Гостиная, ЖК Огни»).
- **Дизайнер** — выбрать пользователя (он будет видеть эту сделку в кабинете).
- Объект клиента, Состав (товары/комплектация), **Сумма**.
- **Статус:** Новая → Заказано у фабрики → Отгружено → Доставлено → Закрыта.
- **Вознаграждение:** Сумма + **Статус вознаграждения** (Начислено / Готово к
  выплате / Выплачено).
- Комментарий, **Вложения** (файлы — приватные, видны дизайнеру в кабинете).

Дизайнер видит статус сделки и вознаграждения в `/cabinet/deals` (только чтение).

## 12. Заявки и подписки

- **Leads** — заявки с сайта (форма «Узнать ц各», звонок, контакты). Поля: имя,
  телефон, e-mail, сообщение, товар, размер, тип, **Статус** (Новая/Обработана).
  Обрабатывайте и меняйте статус.
- **Subscribers** — подписчики «Купона на скидку». У каждого — **промокод**.
  Когда клиент предъявил код в салоне, поставьте **promo_status = Использован**.
