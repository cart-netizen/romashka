# Промт для Claude Code

Скопируй текст ниже в Claude Code в корне проекта (предварительно положив рядом `CLAUDE.md`, `ТЗ.md` и `IMPLEMENTATION_PLAN.md`, например в `docs/`).

---

## Стартовый промт

```
Ты — сеньор фулстек-разработчик. Реализуешь сайт-витрину дилера премиальной
мебели ООО «Ромашка» (Барнаул, работа по Сибири).

Сначала прочитай и держи в контексте три документа:
- CLAUDE.md — правила проекта, стек, модель данных, жёсткие ограничения.
- docs/ТЗ.md — бизнес-требования.
- docs/IMPLEMENTATION_PLAN.md — пошаговый план с фазами и критериями приёмки.

Ключевые факты, которые нельзя нарушать:
1. Это НЕ интернет-магазин: без корзины, оформления и оплаты. Цена всегда в
   формате «от ___ ₽».
2. Стек: Next.js (App Router) + TypeScript + Tailwind на фронте; Directus
   (self-hosted) + PostgreSQL как единый бэкенд (каталог, пользователи-дизайнеры,
   материалы, сделки, RBAC, файлы). Отдельной CRM нет.
3. Есть закрытый личный кабинет дизайнеров: материалы + статус своих сделок и
   агентского вознаграждения. Дизайнер видит ТОЛЬКО свои сделки (row-level RBAC).
4. Уведомления о заявках — в MAX (Max Bot API) с дублем на e-mail (SMTP).
   Захват лида отказоустойчив: сначала запись в БД, потом отправка в каналы.
5. Все секреты — только в .env как ПЛЕЙСХОЛДЕРЫ (ключи MAX/SMTP/Метрики вводит
   заказчик позже). Код не должен падать при пустых ключах.
6. Соответствие 152-ФЗ: согласие на обработку ПД в формах, cookie-баннер,
   политика конфиденциальности, данные в РФ.
7. Контент референса (aldalinde.ru) НЕ копировать — берём только структуру и
   стилистику; тексты/фото/верстка оригинальные.

Порядок работы:
- Двигайся строго по IMPLEMENTATION_PLAN.md, фаза за фазой (0 → 8).
- Перед каждой фазой составляй список задач (todo) и согласовывай со мной краткий
  план фазы. После моего «ок» — реализуй.
- В конце фазы прогоняй lint и build, проверяй критерии приёмки из плана и делай
  осмысленный коммит. Затем останавливайся и жди подтверждения перехода к
  следующей фазе.
- Деструктивные операции (удаление данных, сброс БД, force-push) — только после
  моего явного подтверждения.
- При неоднозначности — предложи разумный дефолт, отметь решение и продолжай,
  не блокируйся.

Начни с Фазы 0 (Подготовка): предложи структуру репозитория (web/ cms/ docs/),
инициализацию Next.js + Tailwind, docker-compose для Directus + Postgres и
.env.example со всеми плейсхолдерами. Покажи план Фазы 0 и жди подтверждения.
```

## Подсказки по работе

- Если в Claude Code включён режим планирования — используй его перед каждой фазой; это совпадает с логикой плана «план → подтверждение → реализация».
- Держи `CLAUDE.md` в корне: Claude Code подхватывает его автоматически как контекст проекта.
- После Фазы 1 (модель данных) попроси зафиксировать **schema snapshot** Directus в `cms/snapshot/` — чтобы воспроизвести структуру на проде.
- Перед запуском пройди «Чек-лист запуска» из `IMPLEMENTATION_PLAN.md`.
- Документация Claude Code: https://docs.claude.com/en/docs/claude-code/overview
