# Техническое задание

## Сайт-витрина дилера премиальной мебели «Ромашка»

**Версия:** 1.1
**Заказчик:** ООО «Ромашка»
**Салон:** г. Барнаул, проспект Космонавтов, 6г
**Регион работы:** Алтайский край, Республика Алтай, Новосибирская область и регионы Сибири

---

## 1. Назначение и цели

### 1.1. Тип проекта
Сайт-визитка с каталогом ассортимента премиальной мебели от 4 мебельных фабрик. **Это не интернет-магазин:** нет корзины, оформления заказа и онлайн-оплаты. Каталог служит витриной, цель которой — привести пользователя к обращению (звонок, сообщение в мессенджер, визит в салон).

Дополнительно к публичной части реализуется **закрытый личный кабинет дизайнеров-партнёров** с доступом к рабочим материалам и отслеживанием статуса своих сделок и агентского вознаграждения.

### 1.2. Бизнес-цели
- Презентовать ассортимент и фабрики-партнёры в премиальном визуальном стиле.
- Получать обращения (лиды) и доводить пользователя до контакта/визита в салон.
- Дать дизайнерам-партнёрам единое окно: материалы + прозрачный статус сделок и вознаграждения.

### 1.3. Целевая аудитория
1. **Частные клиенты** премиум-сегмента — диваны, кровати, кресла, тумбочки; решение принимается оффлайн.
2. **Дизайнеры интерьеров** (партнёры/агенты) — работают с дилером, получают агентское вознаграждение.

---

## 2. Ассортимент и контент

- **Категории:** Диваны, Кровати, Кресла, Тумбочки. Структура расширяемая.
- **Подкатегории:** у каждой категории — свой список подкатегорий (например, Диваны → Угловые / Прямые / Двухместные / Трёхместные / Модульные / Кожаные). Настраивается заказчиком в админке.
- **Фабрики:** 4 партнёра, у каждой логотип, описание и привязанный ассортимент.
- **Объём:** до 100 позиций в каждой категории (ориентировочно до ~400 товаров).
- **Цена:** в формате **«от ___ ₽»**. Точную стоимость пользователь уточняет через обращение.
- **Наполнение:** вручную через админ-панель Directus. Импорт/выгрузки не требуются.

## 4. Функциональные требования — публичная часть

### 4.1. Шапка и навигация (mega-menu)
- Логотип (Playfair Display), горизонтальное меню по категориям, иконки: поиск, избранное, «Кабинет» (вход дизайнера). **Корзины нет.**
- При наведении на категорию (десктоп) или тапе (мобайл) разворачивается **mega-menu**:
  - **Слева:** заголовок категории + вертикальный список подкатегорий (ссылки на `/catalog/[category]/[subcategory]`).
  - **Справа:** до 3 промо-плиток (фото + заголовок + стрелка), например «Новинки», «Хиты коллекции», «Распродажа». Плитки и их ссылки настраиваются в админке.
  - Фон страницы затемняется оверлеем; меню закрывается по уходу курсора/клику вне.
- На мобильном — раскрывающийся бургер с вложенными подкатегориями (аккордеон).
- Липкая шапка.

### 4.2. Главная страница
- Hero: крупное изображение, серифный заголовок-оффер, CTA «Смотреть каталог».
- Плитка категорий (4 категории с изображениями).
- **Интерактивная сцена «Shop the look»** (см. §4.10).
- Блок «Новинки/Бестселлеры» — лента товаров с ценой «от» и кнопкой «В избранное».
- Блок «Наши фабрики» — 4 логотипа со ссылками.
- Блок «О компании» (кратко) + преимущества.
- Блок «Доставка/Самовывоз/Сборка» (иконки, регион Сибирь).
- Форма заявки + контакты салона, карта.
- Акцентные цветные баннеры с CTA.

### 4.3. Каталог и фильтры
- Левый сайдбар с фильтрами:
  - Тумблер «В наличии».
  - **Фабрика:** выбор из фабрик-партнёров.
  - **Подкатегория** (в пределах категории).
  - **Каркас:** Массив / Металл / Фанера.
  - **Обивка:** Букле / Жаккард / Лён / Натуральная кожа / Смесовая ткань / Шенилл.
  - **Цена** (слайдер диапазона).
  - **Цвета** (свотчи).
  - **Размеры:** Ширина / Высота / Глубина (слайдеры, см).
  - Кнопки «Сбросить» / «Применить».
- Сортировка: по возрастанию/убыванию цены, по новизне.
- Сетка карточек (4 колонки на десктопе, адаптив), пагинация, поиск.
- На мобильном — фильтры в выезжающей панели.

### 4.4. Карточка товара
**Левая колонка — галерея:**
- Вертикальная лента миниатюр + крупное основное изображение; клик по миниатюре меняет основное фото; стрелка-«вниз» для прокрутки миниатюр.

**Правая колонка — информация:**
- Хлебные крошки: Главная › Категория › Подкатегория › Товар.
- Название + иконка «В избранное» (сердечко).
- Рейтинг (звёзды) + «N отзывов» (якорь к блоку отзывов).
- **Артикул** (SKU).
- Цена в формате **«от ___ ₽»** (Playfair Display).
- **Выбор размера:** кнопки-варианты (например, 230×84×106 / 317×84×106); выбранный подсвечен.
- USP-баннер с ротацией сообщений (например, «Персональный подбор обивки — отправим образцы»), с точками-переключателями. Сообщения настраиваются в админке.
- **CTA (вместо «В корзину»):** «Узнать цену» — открывает форму заявки с привязкой товара и выбранного размера; дополнительно «Заказать звонок» / «Написать в MAX».
- Срок изготовления (например, «Срок изготовления — от N недель») — поле товара или значение по умолчанию из настроек.
- **Фабрика-производитель** (со ссылкой на страницу фабрики).

**Аккордеоны (раскрывающиеся блоки):**
- **«Материал изделия и уход»** — таблица «характеристика → значение» (Каркас, Обивка, Наполнение сиденья/спинки/подушек/подлокотников, Ножки, Уход и т.д.). Набор строк гибкий (repeater), т.к. отличается по категориям.
- **«Фактические размеры»** — изображения-чертежи с размерами + дисклеймер «Все изделия измерены вручную. Возможна незначительная погрешность 1–3 см» (текст по умолчанию из настроек).
- **«Информация о товаре»** — полное описание модели.

**Под карточкой:**
- **«Рекомендуем также»** — лента похожих товаров.
- **«Отзывы»** — рейтинг, список отзывов, сортировка, пустое состояние. Отзывы **кураторские** (заводит/модерирует администратор), т.к. публичной регистрации на сайте нет.

### 4.5. Карточка товара в сетке каталога
Изображение, название, краткое описание, фабрика, «Размеров доступно: N», свотчи цветов, цена «от», иконка «В избранное».

### 4.6. Формы и лиды
- Типы: заявка с карточки товара (привязка товара и выбранного размера), обратный звонок, форма на контактах.
- Поля: имя, телефон (обязательно), e-mail/сообщение (опц.), чекбокс согласия 152-ФЗ (обязательный).
- Валидация клиент+сервер, антиспам (honeypot + rate limit).
- При отправке: сохранение лида в БД + уведомление в **MAX** + дубль на **e-mail**.

### 4.7. Избранное
Добавление/удаление без регистрации (localStorage), страница `/favorites`.

### 4.8. Виджеты связи
- Плавающая кнопка мессенджера MAX (и опционально телефон).

### 4.9. Виджет «Купон на скидку» и подписка
- Закреплённая вертикальная вкладка «Купон на скидку» у левого края экрана: позиционирование fixed — не двигается при скролле; присутствует на всех страницах публичной части.
- По клику открывается выезжающая панель в правом нижнем углу.
- Панель из двух частей: слева изображение мебели (~40% ширины), справа (~60%) — текст, форма и кнопка.
- Контент: заголовок «Присоединяйтесь к нам»; текст «Будьте первыми, кто узнает о наших эксклюзивных предложениях. А ещё получите дополнительный купон на 5000 рублей на первый заказ.»; поле ввода e-mail или телефона; кнопка «Подписаться»; обязательный чекбокс согласия 152-ФЗ.
- Закрытие по крестику или клику вне панели; факт закрытия запоминается в сессии.
- При отправке: подписчик сохраняется в БД (`subscribers`), администратору уходит уведомление (MAX/e-mail), и подписчику **выдаётся промокод на 5000 ₽** на первый заказ.
- **Выдача промокода:** для каждого подписчика генерируется уникальный код, сохраняется в его записи и доставляется по каналу контакта — если указан e-mail, код приходит письмом (SMTP); если указан телефон, код приходит через **MAX** (бизнес-уведомление по номеру телефона). Код действует офлайн: его называют/предъявляют в салоне, администратор отмечает использование вручную. Канал, факт и дату отправки фиксируем; при сбое канала запись подписчика не теряется.
- *(Альтернатива по согласованию: единый статический промокод вместо уникальных.)*

### 4.10. Интерактивная сцена на главной («Shop the look»)
- **Основной блок просмотра:** крупное фото интерьера; на предметах мебели — пульсирующие маркеры-точки. При наведении на маркер всплывает подсказка: название товара, цена «от ___ ₽», значок «>»; клик ведёт на карточку товара.
- **Под основным блоком — галерея из 5 миниатюр** сцен (в уменьшенном формате), на миниатюрах также видны точки. Клик по миниатюре открывает соответствующую сцену в основном блоке просмотра.
- Сцены, координаты маркеров и привязка к товарам настраиваются в админке (Directus).

## 5. Личный кабинет дизайнера

### 5.1. Авторизация и доступ
- Вход по e-mail/паролю (учётные записи создаёт администратор).
- Роль «Дизайнер»: видит **только свои** сделки и общие материалы; режим только для чтения.
- Защита маршрутов `/cabinet/*`, выход (logout); восстановление пароля — по согласованию.

### 5.2. Раздел «Материалы»
Список материалов (ссылки/файлы): прайс-листы, чертежи/3D, фактуры тканей, условия сотрудничества, маркетинговые материалы; с группировкой по категориям.

### 5.3. Раздел «Сделки»
- Список сделок дизайнера со статусом.
- Модель сделки:
  - Номер/название, объект/клиент, состав, сумма.
  - **Статус:** Новая → Заказано у фабрики → Отгружено → Доставлено/Готово → Закрыта.
  - **Агентское вознаграждение:** сумма + статус (Начислено / Готово к выплате / Выплачено).
  - Комментарий, даты, вложения.
- Заполняется вручную администратором.
- **Реализация без отдельной CRM:** сделки — коллекция в Directus; администратору доступен Kanban-вид (перетаскивание по статусам). Для ≤5 одновременных сделок это оптимально.

## 6. Интеграции

### 6.1. Уведомления в MAX
- Канал: **Max Bot API** (`platform-api.max.ru`, токен в заголовке `Authorization`, доставка через Webhook).
- Юрлицо ООО «Ромашка» зарегистрировано в MAX для бизнеса и верифицировано.
- Назначение: уведомления о новых заявках/подписках администратору; опционально — уведомление дизайнеру при смене статуса сделки.
- **Ключи (токен бота, chat_id) заказчик вводит сам** — в конфигурации предусмотреть плейсхолдеры.

### 6.2. E-mail (резервный канал)
Дублирование заявок/подписок на e-mail через SMTP.

### 6.3. Аналитика
Яндекс.Метрика + Яндекс.Вебмастер. ID/коды — плейсхолдеры.

### 6.4. Карта
Карта салона на странице контактов (Яндекс.Карты, iframe).

## 8. Дизайн и UX

### 8.1. Стилистика (ориентир — референс, но с собственным брендом и контентом)
- Премиальная, тёплая, «редакторская» эстетика.
- **Палитра:** кремовый/слоновая кость (фон), глубокий бордовый и терракотово-коричневый (акценты, баннеры, футер), тёмно-графитовый текст.
- **Типографика (подтверждено по CSS референса):** заголовки, логотип и цены — **Playfair Display** (serif, `--font-playfair-display`); весь текст, навигация, интерфейс и символ валюты — **Montserrat** (`--font-montserrat`). Оба с полной поддержкой кириллицы, подключение через `next/font/google`.
- Много «воздуха», крупные качественные фото товаров и интерьеров.

> **Важно (юридически):** копировать тексты, фотографии и фирменный стиль сайта-референса нельзя. Используется только общая структура и стилистическое направление; контент, копирайтинг, медиа и файлы шрифтов — собственные/из открытых источников.

### 8.2. Адаптивность
Mobile-first, корректное отображение от 360px до широких десктопов, кроссбраузерность (актуальные Chrome, Safari, Firefox, Edge, Яндекс.Браузер).

## 9. Технический стек

| Слой | Технология |
|---|---|
| Frontend | Next.js (App Router) + TypeScript + Tailwind CSS, SSG/ISR |
| CMS / Backend | Directus (self-hosted): каталог, пользователи, материалы, сделки, RBAC, файлы |
| База данных | PostgreSQL |
| Изображения | next/image + трансформации Directus |
| Уведомления | Max Bot API (webhook) + SMTP (nodemailer) |
| Хостинг | VPS в РФ (Selectel/Timeweb), Nginx, Docker Compose |
| Аналитика | Яндекс.Метрика + Вебмастер |

Обоснование: единый бэкенд Directus закрывает каталог, авторизацию, хранение материалов и учёт сделок с разграничением доступа — без дублирования систем и без платной CRM.

## 10. Нефункциональные требования
- **Производительность:** Core Web Vitals в «зелёной» зоне, оптимизация изображений и шрифтов, ISR-кэширование каталога.
- **Безопасность:** HTTPS, защита админки и ЛК, httpOnly-куки для токенов, row-level доступ к данным дизайнеров, серверная валидация форм.
- **152-ФЗ:** хранение ПД в РФ, политика обработки ПД, явное согласие в формах, cookie-баннер.
- **Надёжность:** регулярные бэкапы БД и файлового хранилища.
- **Доступность контента:** заказчик сам ведёт каталог, материалы и сделки через админ-панель.

## 11. Что предоставляет заказчик
- Ключи: токен бота MAX и chat_id, ID Яндекс.Метрики, код верификации Вебмастера, доступ к SMTP *(в коде — плейсхолдеры)*.

- Логотип/фирстиль дилера (если есть).

## 12. Открытые/уточняемые позиции
