# Чек-лист запуска — сайт «Ромашка»

Финальная проверка перед публикацией (ТЗ §7, IMPLEMENTATION_PLAN §7).

## 1. Инфраструктура (VPS, РФ)
- [ ] VPS в РФ, Docker + Docker Compose установлены.
- [ ] DNS: `A`-записи для `<домен>` и `cms.<домен>` → IP сервера.
- [ ] Открыты порты 80/443; firewall настроен; Postgres наружу НЕ открыт.

## 2. Конфигурация
- [ ] `deploy/.env` заполнен (см. `deploy/.env.example`): домены, секреты БД,
      `DIRECTUS_KEY/SECRET` (новые), пароль админа.
- [ ] **Сервисный токен Directus** для серверной записи лидов/подписок:
      создать отдельного пользователя с минимальными правами (create
      `leads`/`subscribers`, чтение `subscribers` для уникальности кода),
      выписать статический токен → `DIRECTUS_ADMIN_TOKEN`. (Не использовать
      админ-токен на проде.)
- [ ] Ключи MAX (`MAX_*`), SMTP (`SMTP_*`, `LEAD_EMAIL_TO`), Метрика
      (`NEXT_PUBLIC_YANDEX_METRICA_ID`), Вебмастер (`YANDEX_WEBMASTER_VERIFICATION`).

## 3. Запуск стека
- [ ] `cd deploy && docker compose -f docker-compose.prod.yml up -d --build`.
- [ ] SSL: выпустить сертификаты (certbot) для `<домен>`, `www`, `cms.<домен>`
      (см. `deploy/README.md`), перезапустить nginx.
- [ ] Применить схему Directus: `directus schema apply snapshot/snapshot.yaml`;
      выполнить `npm run access && npm run presets && npm run seed` (или завести
      контент вручную). Установить Kanban-расширение из Marketplace.

## 4. Контент (через админку Directus)
- [ ] `site_settings`: телефон, e-mail, адрес, часы, мессенджер MAX, карта,
      дисклеймер размеров, дефолтный срок изготовления, сумма промокода.
- [ ] Каталог: товары (фото, чертежи, артикулы, размеры, характеристики),
      фабрики, подкатегории, промо-плитки меню. Фото каталога — в публичной папке.
- [ ] Отзывы, USP-сообщения, showcase-сцены с хотспотами.
- [ ] Учётные записи дизайнеров (роль Designer), сделки, материалы.

## 5. Проверки
- [ ] Тестовая заявка и подписка доходят в MAX и на e-mail; промокод выдаётся
      (e-mail и телефон).
- [ ] Изоляция дизайнеров: чужие сделки недоступны (404), приватные файлы — только
      через кабинет.
- [ ] Mega-menu и интерактивная сцена корректны на десктопе и мобайле.
- [ ] `/sitemap.xml` и `/robots.txt` отдаются; JSON-LD валиден (Rich Results Test).
- [ ] Метрика считает визиты; Вебмастер подтверждён; sitemap отправлен.
- [ ] Lighthouse: Performance/SEO/Best Practices/Accessibility в «зелёной» зоне
      (изображения avif/webp, шрифты `next/font`, скрипты `afterInteractive`).
- [ ] 152-ФЗ: согласие в формах, cookie-баннер, политика конфиденциальности.

## 6. Эксплуатация
- [ ] Бэкапы: `deploy/backup.sh` в cron (БД + файлы), проверено восстановление.
- [ ] HTTPS-редирект, HSTS, заголовки безопасности активны.
- [ ] Мониторинг доступности (uptime), логи nginx/web/directus.
