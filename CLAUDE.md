# CLAUDE.md

Контекст проекта для Claude Code. Читай этот файл перед работой и сверяйся постоянно.

## Что это за проект

Сайт-витрина дилера премиальной мебели **ООО «Ромашка»** (салон в Барнауле, работа по Сибири). Мебель 4 фабрик: диваны, кровати, кресла, тумбочки.

**Это НЕ интернет-магазин.** Нет корзины, оформления заказа и оплаты. Цель — показать ассортимент и привести клиента к обращению (звонок / мессенджер / визит). Цена всегда **«от ___ ₽»**.

Дополнительно — закрытый **личный кабинет дизайнеров-партнёров**: материалы + отслеживание статуса своих сделок и агентского вознаграждения.

Полные требования — `ТЗ.md`. Пошаговый план — `IMPLEMENTATION_PLAN.md`. **Работай строго по плану, фаза за фазой.**

## Технологический стек

- **Frontend:** Next.js (App Router) + TypeScript + Tailwind CSS. SSG/ISR для публичных страниц.
- **CMS/Backend:** Directus (self-hosted) — каталог, пользователи-дизайнеры, материалы, сделки, RBAC, файлы.
- **БД:** PostgreSQL.
- **Уведомления:** Max Bot API (webhook) + SMTP (nodemailer) как резерв.
- **Хостинг (прод):** VPS в РФ, Nginx, Docker Compose. Локально — Docker Compose (Postgres + Directus) и dev-сервер Next.js.

## Модель данных (Directus) — кратко

- `factories` — фабрики (name, slug, logo, description, website, sort, status).
- `categories` — категории (4 шт.) + `subcategories` (name, slug, category→, sort, status).
- `menu_promos` — промо-плитки mega-menu (category→, title, image, link, sort) — Новинки/Хиты/Распродажа.
- `products` — товары: name, slug, **sku**, category→, subcategory→, factory→, **price_from**, descriptions, main_image, gallery, frame, upholstery[], colors[], **characteristics** (repeater label/value), **sizes** (repeater label/Ш×В×Г), **dimensions_images**, lead_time_note, width/height/depth_cm, variants_count, style, is_bestseller/is_new/is_sale/in_stock, sort, status.
- `reviews` — кураторские отзывы (product→, author_name, rating, text, status). Public: read published.
- `usp_messages` — ротация сообщений в USP-баннере карточки.
- `colors` — свотчи (name, hex, sort).
- `leads` — заявки (name, phone, email, message, product→, selected_size, source_page, type, status, created_at).
- `subscribers` — подписки из виджета «Купон на скидку» (contact, consent, source, **promo_code**, promo_channel email/max, promo_sent_at, promo_status issued/redeemed). Public: только create; промокод заполняет сервер.
- `materials` — материалы для дизайнеров (title, description, file/url, category, sort, status).
- `deals` — сделки (number, title, designer→users, client_object, items, amount, **status**, commission_amount, commission_status, comment, attachments, dates).
- `showcase_scenes` / `showcase_hotspots` — «Shop the look»: фото + пульсирующие маркеры (pos_x/pos_y %, product→).
- `site_settings` — синглтон: контакты, адрес, часы, мессенджеры, SEO-дефолты, **dimensions_disclaimer**, **default_lead_time_note**.
- Дизайнеры — это `directus_users` с ролью **Designer**.

Подробные поля и связи — в `IMPLEMENTATION_PLAN.md` §2.

## Роли и доступ (критично)

- **Admin** — заказчик: полный доступ через админку Directus.
- **Designer** — читает **только свои** сделки (`deals.designer = $CURRENT_USER`) и материалы; правит только свой профиль.
- **Public** — читает опубликованные `products/categories/subcategories/menu_promos/factories/colors/reviews/usp_messages/showcase_*/site_settings`; создаёт `leads`, `subscribers`. Больше ничего.

Row-level правила обязательны. Никогда не отдавай данные одного дизайнера другому.

## Ключевые UI-блоки

- **Mega-menu:** наведение на категорию открывает панель — слева подкатегории, справа промо-плитки (`menu_promos`), оверлей-затемнение. Мобайл — вложенный аккордеон.
- **Карточка товара:** слева галерея (верт. миниатюры + основное фото + стрелка). Справа: крошки, название + избранное, рейтинг + «N отзывов», **артикул**, цена «от», **выбор размера** кнопками, USP-баннер с ротацией, CTA **«Узнать цену»** (НЕ «В корзину»), срок изготовления, ссылка на фабрику. Аккордеоны: «Материал изделия и уход» (таблица из `characteristics`), «Фактические размеры» (`dimensions_images` + дисклеймер), «Информация о товаре». Ниже — «Рекомендуем также» и «Отзывы» (кураторские).
- **Интерактивная сцена на главной:** основной просмотр (фото + пульсирующие маркеры) + галерея 5 миниатюр сцен (тоже с точками); клик по миниатюре открывает сцену в основном блоке; клик по маркеру → карточка товара.

## Жёсткие правила (не нарушать)

1. **Никакого e-commerce:** без корзины, оформления, оплаты. Цена всегда «от ___ ₽». CTA карточки — «Узнать цену».
2. **Секреты — только в `.env`** (плейсхолдеры; ключи MAX/SMTP/Метрики вводит заказчик). Код не падает при пустых ключах. См. `.env.example`.
3. **Админ-токен Directus — только на сервере.** Публичное чтение — read-only public-роль / серверный фетч.
4. **Токены дизайнера — в httpOnly-куки.** `/cabinet/*` защищён middleware.
5. **Захват лидов/подписок отказоустойчив:** сначала запись в БД, потом MAX и e-mail в `try/catch`; падение канала не теряет запись.
6. **152-ФЗ:** обязательный чекбокс согласия в формах, cookie-баннер, политика, данные в РФ.
7. **Контент референса (aldalinde.ru) НЕ копировать** — только структура и стилистика. Тексты/фото/верстка/файлы шрифтов — оригинальные.
8. **Кириллица:** все шрифты с поддержкой кириллицы.
9. **Отзывы — кураторские** (заводит администратор), без публичной регистрации.
10. **Промокод 5000 ₽:** при подписке сервер генерирует уникальный код, сохраняет в `subscribers` и доставляет подписчику — e-mail (если e-mail) или MAX по номеру (если телефон). Код действует офлайн в салоне; администратор отмечает использование.

## Стиль кода

**Принцип:** код должен быть чистым, читаемым и не перегруженным. Применяй проверенные паттерны проектирования, best practices и industry-standard подходы в решениях, архитектуре и дизайне. Предпочитай простые поддерживаемые решения переусложнённым; следуй принципам DRY, KISS, SOLID и разделению ответственности; избегай дублирования, лишних абстракций и преждевременной оптимизации.

- TypeScript strict, без лишнего `any`.
- Серверные данные — серверные компоненты / server actions / API routes; Directus с привилегированным токеном не дёргать с клиента.
- Валидация форм — `zod` (клиент + сервер).
- Компоненты небольшие, переиспользуемые; UI-примитивы в одной папке.
- Стили — Tailwind + дизайн-токены, без «магических» значений вразброс.

## Дизайн-токены (ориентир)

- Фон: кремовый/слоновая кость. Текст: тёмно-графитовый.
- Акценты: глубокий бордовый и терракотово-коричневый (баннеры, футер, кнопки).
- **Шрифты (подтверждено по CSS референса):** заголовки/логотип/цены — **Playfair Display** (serif, `--font-playfair-display`); текст/навигация/интерфейс/валюта — **Montserrat** (`--font-montserrat`). Оба с кириллицей, через `next/font/google`.
- Эстетика: премиум, «воздух», крупные фото. Точные значения — `IMPLEMENTATION_PLAN.md` §3.
